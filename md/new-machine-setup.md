# New Machine Setup

Steps to get crate-project running on a different computer.

---

## Prerequisites

Install these first:

- [Git](https://git-scm.com/)
- [Python 3.11+](https://www.python.org/)
- [uv](https://docs.astral.sh/uv/getting-started/installation/) — Python package manager
- [Node.js + npm](https://nodejs.org/)
- [Claude Code](https://claude.ai/code) — `npm install -g @anthropic-ai/claude-code`

**Windows only:** Essentia does not work on native Windows. Install [WSL2](https://learn.microsoft.com/en-us/windows/wsl/install) with Ubuntu and do all project work inside it.

---

## Setup Steps

**1. Clone the repo**

```bash
git clone https://github.com/Ivor-Vice-Bulaja/crate-project.git
cd crate-project
```

**2. Install Python dependencies**

```bash
# Base dependencies (FastAPI, AcoustID, MusicBrainz, etc.)
uv sync

# Essentia + TensorFlow for audio analysis (large — takes several minutes)
uv sync --extra analysis

# Dev tools (pytest, ruff, etc.)
uv sync --extra dev
```

**3. Install frontend dependencies**

```bash
cd frontend && npm install
```

**4. Create your `.env` file**

```bash
cp .env.example .env
```

Then open `.env` and fill in your API keys:

```
ANTHROPIC_API_KEY=
ACOUSTID_API_KEY=
MUSICBRAINZ_APP=CrateApp/0.1 (your@email.com)
DISCOGS_TOKEN=
DB_PATH=./crate.db
MUSIC_FOLDER=/absolute/path/to/your/music
LOG_LEVEL=INFO
```

---

## What Is Not in the Repo

These are gitignored and must be set up manually on each machine:

| Item | Where to get it |
|---|---|
| API keys | Your own accounts (Anthropic, AcoustID, Discogs) |
| Music folder | Copy from your existing machine or external drive |
| `crate.db` | Regenerated by running the import pipeline — not transferable |

---

**3. Download ML models**

The `models/` folder is gitignored and must be populated manually. Models needed:

```
discogs-effnet-bs64-1.pb / .json
danceability-discogs-effnet-1.pb
deam-msd-musicnn-2.pb
msd-musicnn-1.pb
mood_aggressive/happy/party/relaxed/sad-discogs-effnet-1.pb
mtg_jamendo_instrument-discogs-effnet-1.pb / .json
mtg_jamendo_moodtheme-discogs-effnet-1.pb / .json
voice_instrumental-discogs-effnet-1.pb
voice_instrumental-musicnn-msd-2.pb
discogs_track/artist/label/release_embeddings-effnet-bs64-1.pb
```

Download from: https://essentia.upf.edu/models/

Until models are downloaded, run Essentia scripts with `--no-ml`.

---

## Verify Setup

```bash
uv run python -c "import essentia; print('Essentia OK')"
uv run pytest
```

If Essentia fails on Windows, you are not inside WSL2. See the Windows note above.
